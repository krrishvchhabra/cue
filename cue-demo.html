How Cue works

How transaction data flows through Cue's signal extraction, event classification, and action-routing layers — end to end, no core banking changes.

Step 1
Data Ingestion
Every transaction streams in real time via Kafka + Spark Streaming. Historical data is pulled for context per customer.
Step 2
Signal Extraction
Rule-based filters tag each transaction — MCC codes, amount anomalies, recurring patterns, new payee names.
Step 3
Signal Clustering
An XGBoost model groups weak signals per customer and scores them for life-event confidence.
Step 4
Event Classification
Life event labelled with a confidence score. Below the 70% threshold, the event is suppressed — no false positives.
Step 5
Product Mapping
Rule logic + collaborative filtering match the detected event to the right SBI Group product, based on similar customer cohorts.
Step 6
Action Routing
Right product, right channel — RM alert, in-app nudge, or call-centre trigger, chosen by event type and value.
